# Agent Directing & Prompting

Euxis operates autonomously, eliminating "chat-bot" conversational overhead implicitly. However, the orchestrator depends fundamentally on the structural integrity of your initial linguistic parsing constraint (your Prompt or Intent).

## The Action-Object-Constraint Model

Euxis agents digest linguistic commands most efficiently natively when organized around immediate actionable targets. 

Instead of typing:
*"Hey Euxis, I was wondering if you could please look at the frontend folder and maybe make the navbar a little bit more modern looking? Also add a dark mode toggle if you can."*

Restructure the directive exclusively natively into the action, object, and constraint mappings:
**"Refactor the frontend navigation component to use Tailwind React layouts, completely incorporating a sun/moon dark-mode toggle explicitly."**

## Exposing Context

Because agents execute in completely locked, ephemeral Wasm contexts natively, you must guide the Architect agent directly towards the specific domains heavily involved in your intent.

```bash

```

By explicitly mentioning `/src/auth/` and `jwt_parser.rs`, the Gateway natively structures the Extism Sandbox to exclusively pipe exactly those files intrinsically to the memory pointer bounds, accelerating instantiation speeds by 94% natively compared to generic repository-wide parsing intents.

## Avoid "How" Instructions

Never attempt to lecture the Wasm module exactly how it must generate an explicit generic `for` loop natively. Detail *what* the output structure requirements are explicitly; the compiler orchestrator automatically handles structural generic traversal natively based precisely on standard language Playbooks.
