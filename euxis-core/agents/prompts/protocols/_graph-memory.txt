## GRAPH MEMORY PROTOCOL

### Graph Awareness
- The Euxis Graph Memory (`euxis-graph`) maintains a knowledge graph alongside the vector Cortex.
- The graph stores entities, relationships, and provenance chains using NetworkX.

### When to Create Graph Entities
You MUST create graph entities when:
- Discovering a new codebase component (module, service, API endpoint)
- Identifying a causal relationship (bug caused by X, decision led to Y)
- Recording an agent interaction (agent X delegated to agent Y)
- Mapping a dependency (service A depends on service B)

### When to Create Graph Edges
You MUST create graph edges when:
- Two entities have a causal, structural, or temporal relationship
- A memory references another memory or entity
- An agent produces an artifact that another agent consumes

### Entity Types
| Type | Use For | Example |
|------|---------|---------|
| `entity` | Codebase components, concepts | `"auth-module"`, `"JWT-signing"` |
| `memory` | Cortex memory references | `"bug-42-fix"`, `"deploy-v5"` |
| `agent` | Fleet agents | `"architect"`, `"debugger"` |
| `session` | Interaction sessions | `"20260130-133152"` |

### Relation Types
| Relation | Meaning | Example |
|----------|---------|---------|
| `related_to` | General association | auth-module → JWT-signing |
| `produced_by` | Creation provenance | bug-42-fix → debugger |
| `depends_on` | Structural dependency | login-service → auth-module |
| `contradicts` | Conflicting information | finding-A → finding-B |
| `references` | Citation or mention | memory-X → entity-Y |
| `part_of` | Composition | login-endpoint → auth-module |

### Hybrid Recall
When performing complex recall tasks, you SHOULD use both systems:
```bash
# Vector recall for semantic similarity
euxis-cortex recall "<query>" --type procedural
# Graph traversal for relationships
euxis-graph neighbors "<entity>" --depth 2
```

### Rules
- NEVER create duplicate entities. Query first with `euxis-graph query` before adding.
- Entity names MUST be lowercase-kebab-case.
- Every edge MUST use a valid relation type from the table above.
- Graph operations MUST be logged in the audit trail.
