## MESSAGE BUS PROTOCOL

### Bus Awareness
- The Euxis Message Bus (`euxis-bus`) provides async pub/sub communication between agents.
- Messages are JSON envelopes with: sender, topic, priority, payload, timestamp, correlation_id.

### Mandatory Publications
Every agent MUST publish status events to the bus at these lifecycle points:
- **Task Start:** `euxis-bus publish agent-status "<agent_id> started task: <summary>" --sender <agent_id> --priority P2`
- **Task Complete:** `euxis-bus publish agent-status "<agent_id> completed task: <summary>" --sender <agent_id> --priority P2`
- **Task Failure:** `euxis-bus publish agent-status "<agent_id> FAILED task: <summary>. Reason: <reason>" --sender <agent_id> --priority P0`
- **Escalation:** `euxis-bus publish escalations "<description>" --sender <agent_id> --priority <P0|P1>`
- **Conflict Detection:** `euxis-bus publish conflict-alerts "<description>" --sender <agent_id> --priority P1`

### Topic Conventions
| Topic | Purpose | Default Subscribers |
|-------|---------|-------------------|
| `agent-status` | Agent lifecycle events | orchestrator, telemetry |
| `task-events` | Task creation/assignment/completion | orchestrator |
| `memory-updates` | Cortex memory changes | librarian, orchestrator |
| `conflict-alerts` | Inter-agent conflict detection | orchestrator, reviewer |
| `health-checks` | Periodic health status | telemetry |
| `escalations` | Priority escalation requests | orchestrator |

### Rules
- NEVER publish sensitive data (credentials, PII) to any topic.
- P0 events MUST be published immediately, not batched.
- Custom topics are allowed but MUST be registered in euxis-data/bus/topics.json.
- Messages MUST NOT exceed 4096 bytes.
